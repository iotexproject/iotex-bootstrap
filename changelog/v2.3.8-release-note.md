# v2.3.8 Release Note

## Summary

v2.3.8 is a **recommended** release with improvements to the IOSwarm coordinator. All changes are scoped to the `ioswarm/` package.

## Changes

### Fix: gRPC Max Message Size Increased to 32MB

State diff messages can exceed gRPC's default 4MB limit on blocks with large state changes (e.g., contract deployments, batch operations). Affected agents would see:

```
rpc error: code = ResourceExhausted desc = grpc: received message larger than max (6815590 vs. 4194304)
```

This caused the state diff stream to disconnect and reconnect repeatedly, degrading L4 agent accuracy. The fix sets `MaxRecvMsgSize` and `MaxSendMsgSize` to 32MB on the coordinator's gRPC server.

> Note: The agent side (`ioswarm-agent` repo) also needs the same setting on its gRPC dial options.

### Feat: Nonce Race Exclusion for Shadow Accuracy

When two transactions from the same sender are both in the actpool and dispatched to agents, the agent correctly validates both at dispatch time (the nonce matches). However, only one can succeed on-chain — the other fails because its nonce was consumed. This is a nonce race, not an agent error.

`CompareWithActual` now builds a `sender → valid` map per block. If a false positive (agent=valid, actual=invalid) has another transaction from the same sender that succeeded in the same block, and the agent's result has an empty `reject_reason` (distinguishing it from real failures like out-of-gas or reverts), it is classified as a nonce race and excluded from the mismatch count.

A new field `ShadowStats.NonceRaceExcluded` tracks the number of excused false positives.

### Feat: `/api/rewards` Endpoint

New public endpoint `GET /api/rewards?agent=<agent-id>` for querying per-agent reward history. Returns:

```json
{
  "agent_id": "...",
  "history": [
    { "epoch": 100, "amount_iotx": 12.5, "tasks": 40, "accuracy_pct": 100.0, "rank": 3, "bonus_applied": true }
  ],
  "totals": { "earned_iotx": 250.0, "total_tasks": 800, "avg_accuracy": 99.8 }
}
```

The endpoint is exempt from the HMAC auth middleware (alongside `/healthz` and `/api/stats`) so agents can query their own history without credentials.

### Fix: Stale `taskSender` Map Cleanup

A new `taskSender sync.Map` (taskID → sender address) was added to support nonce race detection. It is now cleaned up alongside `txHashToTaskID` during the periodic cleanup that runs every 1000 blocks, preventing unbounded memory growth.

### Fix: Empty Reward History Returns `[]` Instead of `null`

`/api/rewards` now returns an empty JSON array `[]` (not `null`) when an agent has no reward history.
