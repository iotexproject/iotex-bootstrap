# v2.3.5 Release Note

## Summary

v2.3.5 is a **recommended** patch release that introduces height-gated blacklist enforcement. The blacklist added in v2.3.4 is now only enforced starting from a specific activation height, preventing it from affecting transaction validation on historical blocks.


## Key Features and Enhancements

### Height-Gated Blacklist Enforcement
v2.3.5 adds a `BlackListActiveHeight` configuration to control when the blacklist takes effect. On mainnet, the blacklist will be enforced starting from block height **45,404,174**.

- **Activation Height**: The blacklist is only enforced at or after the configured `BlackListActiveHeight`, avoiding interference with historical block validation
- **Backward Compatibility**: When `BlackListActiveHeight` is set to `0`, the blacklist is always enforced (same behavior as v2.3.4)
- **Mainnet Default**: The default activation height for mainnet is set to `45404174`

## Testnet Config Update

The testnet `config.yaml` has been updated with two new lines under `actPool`. If you are upgrading an existing testnet node, please add the following to your `config.yaml`:

```yaml
actPool:
  blackList: []                # <-- new
  blackListActiveHeight: 0     # <-- new
```

## Upgrade Priority

v2.3.5 is a **recommended** release for all nodes that have upgraded to v2.3.4.
